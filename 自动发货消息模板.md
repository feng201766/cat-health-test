# 自动发货消息模板

> 用于：小红书虚拟商品支付后自动发送给用户的消息
> 内容：测试链接 + 注意事项 + 售后说明 + Github Pages部署教程

---

## 推荐模板

```
感谢购买❤
[你的测试链接]

📋 你的方案包含：
• 7 道题，约 2 分钟完成
• 每日分餐表（早中晚睡前各喂多少克）
• 每天行动计划 + 猫饿了乱叫怎么办
• 胖了出减肥方案，瘦了出增重方案
• 计算基于兽医 BCS 体况评分 + RER/MER 热量公式

💡 小提示：
• 计算用的是兽医用公式，但 BCS 是你自己摸、自己看判断的，会有偏差，结果参考就行
• 链接永久有效，可重复测试
• 打不开就复制链接到浏览器打开

❓ 做完测试后：

📌 说明：以上计算结果基于兽医通用的 BCS 体况评分和 RER/MER 热量公式，是一次性的初步估算，只能帮你确定一个大致的起点。

要拿到真正精准、适合你家猫的方案，需要持续记录每周的体重变化和 BCS 体况测量，让算法根据实际数据不断修正喂食量，越用越准。

搜索微信小程序「猫体重管理-木子」，每周花 1 分钟记录体重、自测 BCS，让算法帮你持续校准。

【售后】
• 虚拟商品，发货后不支持退换
• 下单后自动发送链接，没收到或有问题请私信
```

---

## 设置方法

1. 打开小红书 → 创作者中心 → 商品管理
2. 找到你的虚拟商品，点"编辑"
3. 发货方式选 **"自动发货（虚拟商品）"**
4. 把上面模板粘贴进去，把 `[你的链接]` 换成实际测试链接
5. 保存

## 如何部署到 GitHub Pages（永久链接）

下面每一步都配了截图区域，操作一遍大概 5 分钟。

### 第 1 步：注册 GitHub 账号

到 https://github.com 注册，用邮箱就行。

### 第 2 步：创建仓库

1. 登录后点右上角 **「+」→「New repository」**
2. 仓库名填 `cat-health-test`
3. 选 **Public**（必须公开，GitHub Pages 免费版不支持私仓）
4. 勾选 **「Add a README file」**（方便后续上传）
5. 点 **「Create repository」**

### 第 3 步：上传 H5 文件

1. 进入你刚创建的仓库，点 **「Add file」→「Upload files」**
2. 把 `cat-health-test.html` 拖进去
3. 点 **「Commit changes」**

### 第 4 步：开启 GitHub Pages

1. 在仓库页面点 **「Settings」**（顶部导航栏）
2. 左侧菜单找到 **「Pages」**
3. 「Branch」下拉菜单选 **main**，文件夹选 **/ (root)**，点 **Save**
4. 等 1-2 分钟，页面刷新后会显示：
   > Your site is live at `https://你的用户名.github.io/cat-health-test/cat-health-test.html`

### 第 5 步：验证链接

复制那个链接，在手机上用浏览器打开，走一遍完整流程确认没问题。

把链接填入自动发货模板的 `[你的链接]` 位置即可。

## 备用部署方案

### 方案 B：Gitee Pages（国内访问快，但需实名）

1. 注册 https://gitee.com
2. 创建仓库 → 上传 `cat-health-test.html`
3. 进入「服务」→「Gitee Pages」→ 开启
4. 获得链接：`https://你的用户名.gitee.io/cat-health-test/cat-health-test.html`

### 方案 C：Cloudflare Pages（全球 CDN，免费）

1. 注册 https://dash.cloudflare.com
2. 进入「Workers & Pages」→「Create」→「Pages」
3. 选「Upload assets」→ 上传 `cat-health-test.html`
4. 项目名填 `cat-health-test` → 创建
5. 获得链接：`https://cat-health-test.pages.dev`
